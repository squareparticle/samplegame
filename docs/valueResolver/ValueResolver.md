# ValueResolver

`ValueResolver` resolves procedural `@` value commands before normal object loading runs.

It is used by map entries, spawner entries, `Object.load`, `Object.update`, and other object-descriptor systems that need authored JSON to produce runtime values.

## Core idea

A JSON value can be literal:

```json
{
    "x": 100,
    "y": 200
}
```

or procedural:

```json
{
    "x": { "@range": { "min": 100, "max": 300 } },
    "y": { "@range": { "min": 400, "max": 500 } }
}
```

Before `LoadComponent` or `Object.update` receives the payload, `ValueResolver` converts procedural commands into plain values.

```text
Object descriptor
  -> ValueResolver resolves @ commands
  -> LoadComponent.load(...) or Object.update
  -> components receive plain values
```

## `@range`

Creates a random number between `min` and `max`.

```json
{ "@range": { "min": 0, "max": 100 } }
```

Example inside a named load payload:

```json
"load": {
    "scale": {
        "x": { "@range": { "min": 0.07, "max": 0.10 } },
        "y": { "@range": { "min": 0.07, "max": 0.10 } }
    }
}
```

## `@intRange`

Creates a random integer by flooring a resolved range value.

```json
{ "@intRange": { "min": 1, "max": 5 } }
```

Good for:

```text
random index values
small integer counts
tile choices
simple difficulty levels
```

## `@pointRange`

Creates a random `{ x, y }` point.

```json
{
    "@pointRange": {
        "x": { "min": 100, "max": 300 },
        "y": { "min": 400, "max": 500 }
    }
}
```

Example:

```json
"load": {
    "position": {
        "@pointRange": {
            "x": { "min": 100, "max": 300 },
            "y": { "min": 400, "max": 500 }
        }
    }
}
```

## `@pick`

Chooses one value from an array.

```json
{ "@pick": ["PENCIL", "GLUE", "SCISSORS"] }
```

Example:

```json
"load": {
    "itemType": { "@pick": ["PENCIL", "GLUE", "SCISSORS"] }
}
```

The selected value is resolved too, so entries can contain procedural values.

```json
{
    "@pick": [
        { "@range": { "min": 0, "max": 100 } },
        { "@range": { "min": 500, "max": 600 } }
    ]
}
```

## `@weightedPick`

Chooses a value using weights.

### Array form

```json
{
    "@weightedPick": [
        { "weight": 5, "value": "PENCIL" },
        { "weight": 2, "value": "GLUE" },
        { "weight": 1, "value": "SCISSORS" }
    ]
}
```

### Object shorthand

```json
{
    "@weightedPick": {
        "PENCIL": 5,
        "GLUE": 2,
        "SCISSORS": 1
    }
}
```

Use array form when values are objects or when you want room for more fields later.

Use object shorthand for simple string choices.

Example inside `load`:

```json
"load": {
    "itemType": {
        "@weightedPick": [
            { "weight": 5, "value": "PENCIL" },
            { "weight": 2, "value": "GLUE" },
            { "weight": 1, "value": "SCISSORS" }
        ]
    }
}
```

## `@loadingRule`

Asks a loading rule for a value.

Loading rules are provided by components or by the current loading context.

### Current component rule

In `EntitySpawnerComponent`, this is used to pass the spawner's generated position into the spawned Entity:

```json
{ "@loadingRule": { "this": "spawnPosition" } }
```

Example:

```json
"load": {
    "position": { "@loadingRule": { "this": "spawnPosition" } }
}
```

Meaning:

```text
Use the spawn position generated by the current spawner.
```

### Type-level rule

A future component can expose a type-level rule:

```json
{ "@loadingRule": { "BalloonComponent": "randomType" } }
```

Meaning:

```text
Ask BalloonComponent for its randomType loading rule.
```

### Local component rule

A specific component instance can expose a local rule:

```json
{ "@loadingRule": { "#BalloonBrain": "randomType" } }
```

Meaning:

```text
Ask the local component with id BalloonBrain for its randomType loading rule.
```

### Long form with properties

Use the long form when the rule needs input properties:

```json
{
    "@loadingRule": {
        "BalloonComponent": {
            "rule": "randomType",
            "properties": {
                "allowSpy": false
            }
        }
    }
}
```

## Nested commands

Procedural commands can be nested inside arrays and objects.

```json
"load": {
    "position": { "@loadingRule": { "this": "spawnPosition" } },
    "scale": {
        "x": { "@range": { "min": 0.07, "max": 0.10 } },
        "y": { "@range": { "min": 0.07, "max": 0.10 } }
    },
    "rotation": { "@range": { "min": 0, "max": 6.283185307179586 } },
    "velocity": {
        "x": { "@range": { "min": -120, "max": 120 } },
        "y": { "@range": { "min": -120, "max": 120 } }
    }
}
```

`LoadComponent` receives the resolved result as plain values.

## Example: procedural map entry

```json
{
    "Object": {
        "entity": "Balloons",
        "load": {
            "position": {
                "@pointRange": {
                    "x": { "min": 100, "max": 150 },
                    "y": { "min": 400, "max": 450 }
                }
            },
            "balloonType": {
                "@pick": ["party", "weather", "spy"]
            }
        }
    }
}
```

## Example: procedural spawner entry

```json
{
    "weight": 4,
    "groups": ["debris", "gameObjects"],
    "Object": {
        "entity": "Debris",
        "load": {
            "position": { "@loadingRule": { "this": "spawnPosition" } },
            "scale": {
                "x": { "@range": { "min": 0.07, "max": 0.10 } },
                "y": { "@range": { "min": 0.07, "max": 0.10 } }
            },
            "rotation": { "@range": { "min": 0, "max": 6.283185307179586 } },
            "velocity": {
                "x": { "@range": { "min": -120, "max": 120 } },
                "y": { "@range": { "min": -120, "max": 120 } }
            }
        }
    }
}
```

## Backward-compatible aliases

The prototype aliases are still supported for now:

```json
{ "@componentRule": "spawnPosition" }
```

```json
{ "@context": "spawnPosition" }
```

Public examples should use:

```json
{ "@loadingRule": { "this": "spawnPosition" } }
```

## Recommended use

Use `ValueResolver` commands when the authored JSON needs runtime variation.

```text
@range
  random numeric values

@intRange
  random integer values

@pointRange
  random positions or points

@pick
  simple random choices

@weightedPick
  random choices with different probabilities

@loadingRule
  values generated by a component or loading context
```

Avoid adding game-state-dependent commands until the loading/query contract is stable. For example, a future `@condition` command should be added only after query-style `execute()` results are clearly defined.

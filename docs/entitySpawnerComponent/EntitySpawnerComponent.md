# EntitySpawnerComponent

`EntitySpawnerComponent` is a runtime object factory.

It creates GameObjects from normal Entity JSON, then applies the same `Object.load` and `Object.update` rules used by map files. This lets maps, spawners, and future editor/AI tooling all share the same object-authoring pipeline.

## Core idea

A spawner entry is split into two parts:

```text
Spawner metadata
  weight
  groups
  idPrefix

Object descriptor
  entity
  load
  update
  execute
```

Example:

```json
{
    "weight": 4,
    "groups": ["debris", "gameObjects"],
    "Object": {
        "entity": "Debris",
        "load": {
            "position": { "@loadingRule": { "this": "spawnPosition" } },
            "scale": { "x": 0.1, "y": 0.1 }
        }
    }
}
```

## Public authoring rule

Put normal object construction fields inside `Object`:

```json
"Object": {
    "entity": "Debris",
    "load": {},
    "update": {},
    "execute": {}
}
```

Put spawner metadata outside `Object`:

```json
{
    "weight": 1,
    "groups": ["debris", "gameObjects"],
    "Object": {}
}
```

This keeps spawner entries close to the same shape used by `map.json`.

## `spawnArea`

`spawnArea` tells the spawner where candidate positions come from.

### World rectangle

```json
"spawnArea": {
    "space": "world",
    "rectangle": {
        "origin": "topLeft",
        "width": 1280,
        "height": 720,
        "padding": 90
    }
}
```

Meaning:

```text
Use the world rectangle from x=0..1280 and y=0..720.
Keep spawned objects at least 90 pixels away from the rectangle edge.
```

### Local circle

```json
"spawnArea": {
    "space": "local",
    "circle": {
        "radius": 500,
        "innerRadius": 100
    }
}
```

Meaning:

```text
Use a ring centered on the spawner object's Transform.
```

Default `space` is `local`.

## Distance filters

Use these to reject bad spawn candidates:

```json
"minDistanceFromTransform": 200
```

Minimum distance from the spawner object's own Transform.

```json
"minDistanceFromGroupEntities": 180,
"avoidGroups": ["playerObject"]
```

Minimum distance from objects in the listed groups.

## Static spawning

`static` creates an initial population once.

```json
"static": {
    "count": 10
}
```

Use this for:

```text
grass
rocks
debris fields
initial enemy placement
initial collectable placement
```

## Dynamic spawning

`dynamic` maintains a live population over time.

```json
"dynamic": {
    "targetCount": 8,
    "burstCount": 2,
    "interval": 3
}
```

Meaning:

```text
Every 3 seconds:
  if fewer than 8 tracked objects are alive:
    spawn up to 2 more
```

Use this for:

```text
enemy populations
meteor showers
ambient traffic
replenishing collectables
```

## Weighted entity entries

`entities` is a weighted object recipe table.

The entries do not need to use different Entity files. Multiple entries can use the same Entity with different load/update values.

```json
"entities": [
    {
        "weight": 4,
        "groups": ["debris", "gameObjects"],
        "Object": {
            "entity": "Debris",
            "load": {
                "position": { "@loadingRule": { "this": "spawnPosition" } },
                "scale": { "x": 0.08, "y": 0.08 }
            }
        }
    },
    {
        "weight": 1,
        "groups": ["debris", "gameObjects"],
        "Object": {
            "entity": "Debris",
            "load": {
                "position": { "@loadingRule": { "this": "spawnPosition" } },
                "scale": { "x": 0.16, "y": 0.16 }
            }
        }
    }
]
```

This produces smaller debris more often than larger debris.

## `@loadingRule` and spawn position

The spawner generates a valid position using `spawnArea` and distance filters. To pass that generated position into the spawned Entity's `LoadComponent`, use:

```json
{ "@loadingRule": { "this": "spawnPosition" } }
```

Example:

```json
"load": {
    "position": { "@loadingRule": { "this": "spawnPosition" } }
}
```

This means:

```text
Use the spawn position generated by the current EntitySpawnerComponent.
```

## Procedural values

Spawner object descriptors can use procedural value commands before `load` or `update` runs.

### `@range`

```json
"rotation": { "@range": { "min": 0, "max": 6.283185307179586 } }
```

### Example inside load

```json
"load": {
    "scale": {
        "x": { "@range": { "min": 0.07, "max": 0.10 } },
        "y": { "@range": { "min": 0.07, "max": 0.10 } }
    }
}
```

## LoadComponent vs Object.update

Prefer `LoadComponent` for normal Entity setup.

```json
"load": {
    "position": { "@loadingRule": { "this": "spawnPosition" } },
    "scale": { "x": 0.1, "y": 0.1 },
    "rotation": 0,
    "velocity": { "x": 40, "y": -20 }
}
```

Use `Object.update` for direct one-off overrides.

```json
"update": {
    "component": {
        "#Motion": {
            "options.torque": { "@range": { "min": -30, "max": 30 } }
        }
    }
}
```

Recommended public pattern:

```text
LoadComponent handles normal setup.
Object.update handles one-off direct overrides.
```

## Manual recipes

`recipes` are named spawn profiles that can be triggered by sequence/component commands.

```json
"recipes": {
    "CrystalReward": {
        "entities": [
            {
                "weight": 1,
                "groups": ["collectables", "gameObjects"],
                "Object": {
                    "entity": "Crystal",
                    "load": {
                        "position": { "@loadingRule": { "this": "spawnPosition" } }
                    }
                }
            }
        ]
    }
}
```

A sequence can trigger:

```json
{
    "@COMPONENT": {
        "id": "RewardSpawner",
        "spawn": { "recipe": "CrystalReward", "count": 3 }
    }
}
```

## Backward-compatible single-entity shorthand

Simple cases may still use a single `entity` field:

```json
"entity": "Debris",
"groups": ["debris", "gameObjects"],
"static": { "count": 6 }
```

For public examples, prefer `entities[]` because it teaches the full object-recipe system.
